# 供应链管理桌面客户端

面向中小制造企业的采购、仓管和销售人员协同处理订单的桌面管理系统。基于 Electron + React + TypeScript + Ant Design 构建。

## 功能模块

### 1. 采购计划 ([PurchaseWindow.tsx](src/components/PurchaseWindow.tsx))
- 创建采购需求，填写物品明细、需求日期、申请人等信息
- 采购单全流程状态管理：草稿 → 待报价 → 已报价 → 已确认 → 部分入库 → 已完成
- 确认交期功能，标记订单紧急程度
- 统计看板：订单总数、待处理、执行中、月度采购金额

### 2. 供应商报价 ([QuoteWindow.tsx](src/components/QuoteWindow.tsx))
- **比价视图**：同一采购单多家供应商报价横向对比，自动识别最低价
- 录入供应商报价明细（单价、交期、运费、税额）
- 采纳最优报价，自动回写采购单确认信息
- 供应商报价状态：待提交 → 已提交 → 已采纳 / 已拒绝
- 统计：节支率分析、平均节支金额

### 3. 入库验收 ([ReceiptWindow.tsx](src/components/ReceiptWindow.tsx))
- 关联采购单登记到货数量
- 质检记录：合格数量 / 不合格数量 / 质检备注
- 质检结果：全部合格 / 部分合格 / 全部不合格
- **自动更新**：库存数量、加权平均成本、库存总值
- 自动生成：库存变动流水、供应商应付账单
- 采购单状态自动更新（部分入库 / 完成）

### 4. 库存台账 ([StockWindow.tsx](src/components/StockWindow.tsx))
- **缺货风险看板**：按紧急/预警等级展示，关联影响的销售订单
- 实时库存查询：物料名称/SKU/分类/仓库/风险等级多维度筛选
- 库存健康度：可用库存 / 安全库存 百分比进度条
- **库存分配**：为待分配的销售订单分配可用库存，自动锁定
- 库存流水：完整记录入库/出库/分配/释放等所有变动
- 统计：SKU数、总件数、总价值、缺货预警数量

### 5. 发货跟踪 ([ShippingWindow.tsx](src/components/ShippingWindow.tsx))
- 生成发货单：从已分配库存的销售订单创建
- 自动扣减实际库存，更新销售订单发货状态
- 物流状态推进：已创建 → 拣货中 → 已打包 → 已发货 → 运输中 → 派送中 → 已签收
- **物流时间线**：Timeline 组件展示完整物流节点
- 承运商/运单号管理，预计送达与逾期提醒
- 统计：在途单量、准时送达率、月度发货金额

### 6. 对账中心 ([ReconciliationWindow.tsx](src/components/ReconciliationWindow.tsx))
- **按供应商汇总**：未结款项、账单数、逾期金额
- 应付账单：付款进度条、到期日预警、逾期红色高亮
- 登记付款：金额 / 方式（银行转账、承兑、现金等）/ 流水号
- **月度采购报表**（6个月趋势）：
  - 采购金额 vs 入库金额柱状图 + 准时交期折线
  - 采购单数 / 物料条目数趋势
  - 供应商采购占比饼图
  - 明细数据表格
- **库存周转报表**：
  - 周转率（次/月）、周转天数计算
  - TOP10 高周转物料柱状图
  - 呆滞物料预警列表
  - 期初/期末库存、出入库金额明细

## 核心业务流

```
采购计划 ──► 供应商报价(比价) ──► 采购确认
                                    │
                                    ▼
销售订单 ──► 库存分配 ──► 生成发货单 ──► 物流跟踪 ──► 客户签收
   │            │
   │            ▼
   └─────► 缺货风险标记(自动)
                                   
入库验收(质检+入库) ──► 库存更新(自动) ──► 应付账单(自动)
                                    │
                                    ▼
                           对账中心 ──► 付款登记
```

## 数据自动联动

| 操作 | 自动触发 |
|------|----------|
| 采纳报价 | 采购单→已确认，回写单价/供应商 |
| 入库验收 | 库存数量/总值↑，库存流水，应付账单生成，采购单状态更新 |
| 库存分配 | 已分配库存↑，可用库存↓，分配流水，销售订单状态更新 |
| 生成发货单 | 实际库存↓，分配库存释放，出库流水，销售发货状态更新 |
| 付款登记 | 已付款/未付款金额更新，账单状态变化 |

## 快速开始

### 安装依赖
```bash
npm install
```

### 开发模式（Vite + Electron 同时启动）
```bash
npm run dev
```

### 构建生产版本
```bash
npm run build
```

### 启动构建后的应用
```bash
npm start
```

## 角色使用场景

### 👔 采购人员
1. 在「采购计划」创建采购需求
2. 在「供应商报价」录入多家报价并进行比价
3. 点击「采纳」确认供应商和价格
4. 在「对账中心」查看应付账单进度

### 📦 仓管人员
1. 到货后在「入库验收」登记数量和质检结果
2. 系统自动更新库存台账
3. 在「库存台账」-「缺货风险看板」处理库存预警
4. 根据销售订单分配库存
5. 发货时在「发货跟踪」创建发货单，推进物流状态

### 💼 销售人员
1. 在「库存台账」确认订单库存分配情况
2. 在「发货跟踪」实时追踪订单物流进度
3. 确认客户签收

### 📊 管理人员
1. 在「对账中心-月度采购报表」分析采购趋势
2. 在「对账中心-库存周转报表」识别呆滞物料，优化库存结构
3. 查看顶部通知栏的实时缺货预警

## Mock 数据说明

项目内置了丰富的演示数据，无需后端即可体验完整流程：
- 10 种物料（管道配件、阀门、管材、仪器仪表、机电设备 5 大类）
- 5 家供应商（含信用评级、付款条款）
- 5 张采购单（覆盖草稿→完成全状态）
- 3 份供应商报价（2家比价）
- 2 张入库单（含部分不合格场景）
- 10 条库存记录 + 7 条库存变动流水
- 3 张销售订单（含部分缺货场景）
- 2 张发货单（含已签收、运输中）
- 3 张应付账单（覆盖未付/部分/已结清）
- 5 个 SKU 库存风险预警

可直接体验：报价采纳 → 入库 → 库存分配 → 发货 → 对账 全流程闭环。
